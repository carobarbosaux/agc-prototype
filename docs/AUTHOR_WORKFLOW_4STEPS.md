# Author Workflow: Subject Creation Flow (4 Steps)
## New Component Structure for PantallaCrearAsignatura

---

## Overview

The **subject creation workflow for Author role** has been restructured into **4 distinct steps**, each with its own component file for precise UI control and maintenance.

**Use case:** Author is assigned a new subject (e.g., Deep Learning) with no content created yet. The workflow guides the Author through academic metadata, contextual information, and summary generation before entering Canvas.

**Reset behavior:** This workflow reinitializes on every page reload (illustrative prototype behavior).

---

## Workflow Structure

### 📍 Entry Point
- Author views assigned subject in profile: **Deep Learning**
- Status: **NOT CREATED** (assignment only, no content exists)
- Author selects subject → navigates to `PantallaCrearAsignatura1.jsx`

---

## Step-by-Step Flow

### **STEP 1: Academic Metadata (Fixed Information)**

**Component:** `PantallaCrearAsignatura1.jsx`

**Purpose:** Display pre-filled academic information that the Author cannot edit (provided by coordinator/system)

**Content displayed (read-only):**
- Subject name: "Deep Learning"
- Degree: "Master in Artificial Intelligence"
- Study type: "Master"
- Knowledge area: "Technology"
- Credits: "6 ECTS"
- Coordinator: "Carlos Mendoza"
- Specialist: "Antonio Abad"
- Model: "4P"

**User action:**
- Button: `[Next]` → proceeds to Step 2

**UI notes:**
- Read-only fields (no editable inputs)
- Clear visual hierarchy
- Shows all required academic context

---

### **STEP 2: Memory Descriptor + Learning Outcomes (Contextual Information)**

**Component:** `PantallaCrearAsignatura2.jsx`

**Purpose:** Author provides contextual information that will serve as the foundation for AI-generated content

**Author completes (editable fields):**

| Field | Type | Options/Constraints |
|-------|------|-------------------|
| Prior knowledge level | Dropdown | Initial \| Intermediate \| Advanced \| Expert |
| Number of topics | Number input | Min: 1, Max: 10 (recommended: 8) |
| Subject approach | Dropdown | Theoretical \| Practical \| Theoretical-practical \| Case-based \| Project-based |
| Mandatory topics/concepts | Text area (open field) | Free text — Topics/concepts that MUST appear in the index |
| Commission options | Checkboxes | ☐ Video plan \| ☐ In-depth section (Apartado a fondo) |
| Reference files | File upload | Accepts: Bibliography \| Documents \| Images |

**Available actions:**
- `[Save]` — Save current progress (draft)
- `[Upload files]` — File management for reference materials
- `[AI Assistant]` — Contextual AI help
- `[GENERATE SUBJECT SUMMARY]` — Main CTA button → proceeds to Step 3

**UI notes:**
- **Shares UI design with Step 1** (same layout, styling, structure)
- All fields are editable
- File upload should support drag-and-drop
- AI Assistant is contextual to current form state

---

### **STEP 3: Subject Summary (Generated Preview)**

**Component:** `PantallaCrearAsignatura3.jsx`

**Purpose:** Display AI-generated summary based on Steps 1-2 data; allow Author to review and refine before generating index

**Content displayed (AI-generated, initially read-only):**
- General subject summary / content approach
- Per-topic explanations: "In this topic we will cover…"
- Topic structure details: headings, highlights, visual elements

**Available actions:**
- `[Save]` — Save the approved summary
- `[Edit summary]` — Modify generated content inline
- `[Go back to edit form]` — Return to Step 2 to re-generate the summary
- `[Give instructions to AI]` — Contextual chatbar for refinement requests
- `[GENERATE INDEX]` — Main CTA button → proceeds to Step 4 (Canvas)

**UI notes:**
- Summary content is generated by simulated AI
- Edit functionality allows inline modifications
- "Go back to edit form" discards current summary and resets Step 2 for re-generation
- AI Assistant chatbar appears below the summary for refinement requests

---

### **STEP 4: Canvas — Index Section**

**Component:** `PantallaCanvas.jsx` (Index section)

**Purpose:** Display generated index and begin content editing workflow in Canvas

**Destination:** When Author clicks `[GENERATE INDEX]` in Step 3, they enter the Canvas with the Index section populated and ready for editing.

**Scope:** From this point, the Author works within the Canvas editor (existing functionality).

### Navigation Management

- **App.jsx state update:** Add `crearAsignaturaStep: 1 | 2 | 3` to track current step
- **Route logic:** When `pantalla === 'crearAsignatura'`, render appropriate step component based on `crearAsignaturaStep`
- **Navigation:** Each component handles its own button actions to update `crearAsignaturaStep`

## Data Flow

### Input → Processing → Output

**Step 1 (Metadata):** 
- Input: Pre-filled academic data from system
- Output: Confirmation to proceed

**Step 2 (Context):**
- Input: Author-provided contextual information
- Output: Form data passed to AI (simulated) → generates Step 3

**Step 3 (Summary):**
- Input: AI-generated summary (from Step 2 data)
- Output: Approved summary passed to Canvas → generates Index

**Step 4 (Canvas):**
- Input: Generated Index
- Output: Canvas Index section ready for editing

---

## Reset Behavior

**Important:** This workflow reinitializes on every page reload/new session.

- User reloads page or logs in again
- Assignment reappears as "NOT CREATED"
- Workflow resets to Step 1
- Any unsaved progress is lost (illustrative prototype behavior)

---

## UI Design Notes

### Shared UI (Steps 1 & 2):
- Same layout structure
- Same typography, spacing, colors
- Consistent form styling
- Allows for precise design changes across both steps

### Step 3 (Summary):
- Read-only display initially (for review)
- Edit mode can be toggled (inline editing)
- Conversational AI chatbar below summary
- "Go back" option to restart from Step 2

### Accessibility:
- Clear button labels (CTA buttons are large, primary color)
- Required fields marked
- Error states for validation
- Progress indicator (optional): "Step 2 of 4"
